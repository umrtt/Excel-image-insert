## 🏗️ Architecture Layers

### Layer 1: Frontend (Web)
- **Technology:** HTML5, CSS3, JavaScript
- **Framework:** Eel (Electron-like wrapper)
- **Responsibility:** User interface and interaction
- **Files:** `web/index.html`, `web/styles.css`, `web/app.js`

### Layer 2: Application Layer (Controllers)
- **Technology:** Python
- **Pattern:** Controller/API Endpoint pattern
- **Responsibility:** Coordinate services and expose APIs to frontend
- **File:** `app/controllers/app_controller.py`
- **Principle:** Single Responsibility - coordinates between services

### Layer 3: Business Logic (Services)
- **Technology:** Python
- **Pattern:** Service layer pattern
- **Responsibility:** Implement business rules and logic
- **Files:** `app/services/image_service.py`, `app/services/excel_service.py`
- **Principle:** Depends on interfaces, not concrete implementations

### Layer 4: Domain Layer (Models & Interfaces)
- **Technology:** Python
- **Pattern:** Domain Model pattern + Interface pattern
- **Responsibility:** Define domain models and abstractions
- **Files:** `app/models/*`, `app/interfaces/*`
- **Principle:** Core business concepts and contracts

### Layer 5: Infrastructure Layer (Handlers)
- **Technology:** Python + xlwings + tkinter
- **Pattern:** Concrete implementation pattern
- **Responsibility:** Implement interfaces with external libraries
- **Files:** `infrastructure/excel_handler.py`, `infrastructure/file_handler.py`
- **Principle:** Dependency Inversion - implement interfaces

### Layer 6: Utilities
- **Technology:** Python
- **Responsibility:** Cross-cutting concerns
- **Files:** `app/utils/logger.py`, `app/utils/validator.py`, `app/utils/file_utils.py`
- **Principle:** Single Responsibility - each utility does one thing

## 💡 SOLID Principles Implementation

### 1. Single Responsibility Principle (SRP)
Each class has **exactly one reason to change**:

```
❌ Bad: class ExcelImageProcessor
  ├─ Opens Excel files
  ├─ Inserts images
  ├─ Logs operations
  ├─ Validates input
  └─ Handles UI updates

✅ Good: Separated into
  ├─ ExcelService - Excel operations
  ├─ ImageService - Image operations
  ├─ Logger - Logging
  ├─ Validator - Validation
  └─ AppController - Coordination
```

### 2. Open/Closed Principle (OCP)
Code is **open for extension, closed for modification**:

```
✅ Example: Adding new image format
  ├─ Update Validator.validate_image_file()
  ├─ Update file_handler.select_files()
  └─ No changes to core logic!
```

### 3. Liskov Substitution Principle (LSP)
All implementations **correctly fulfill the contract**:

```python
# Both can be used interchangeably
handler1: IExcelHandler = ExcelHandler()
handler2: IExcelHandler = OpenpyxlHandler()  # Future alternative
```

### 4. Interface Segregation Principle (ISP)
Clients depend on **specific, small interfaces**:

```
❌ Bad: class IHandler
  ├─ Excel operations (10 methods)
  ├─ File operations (5 methods)
  ├─ Logging operations (5 methods)
  └─ Validation operations (5 methods)

✅ Good: Segregated into
  ├─ IExcelHandler
  ├─ IFileHandler
  ├─ ILogger
  └─ (Validation in utils)
```

### 5. Dependency Inversion Principle (DIP)
High-level modules depend on **abstractions, not concrete implementations**:

```python
# ✅ Good: Service depends on interface
class ExcelService:
    def __init__(self, excel_handler: IExcelHandler):
        self.excel_handler = excel_handler
        # Can use any implementation!

# ❌ Bad: Service depends on concrete class
class ExcelService:
    def __init__(self):
        self.excel_handler = ExcelHandler()  # Tightly coupled!
```

## 📊 Data Flow

### Image Insertion Flow

```
User Interface
    ↓ (Eel call)
    ↓ eel.insert_images()
    ↓
AppController.insert_images()
    ↓
    ├─→ ExcelService.is_file_open() ✓
    ├─→ ImageService.get_image_count() ✓
    └─→ ExcelService.insert_images_batch()
            ↓
            ├─→ Validate each operation
            ├─→ ExcelHandler.insert_image()
            │     ├─→ Get sheet reference
            │     ├─→ Check for merged cells
            │     ├─→ Calculate dimensions
            │     └─→ xlwings: sheet.pictures.add()
            │
            └─→ ExcelHandler.save_workbook()
                    ↓
            ✅ Success Response
                    ↓
            JavaScript Frontend
                    ↓
            Display status message
```

## 📈 Extension Points

### Easy to Add:
1. **New image format support** → Update Validator
2. **Alternative Excel library** → Implement IExcelHandler
3. **Different file dialogs** → Implement IFileHandler

2. Public APIs (with careful versioning)
